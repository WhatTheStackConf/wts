# Issue Workflow

Use this when filing, triaging, planning, or publishing work to the issue tracker.

- Issues and PRDs are tracked in GitHub Issues for `WhatTheStackConf/wts`.
- External PRs are not a triage surface unless the user explicitly asks to review or triage a PR.
- Use the canonical triage labels `needs-triage`, `needs-info`, `ready-for-agent`, `ready-for-human`, and `wontfix`.
- Read [docs/agents/issue-tracker.md](../agents/issue-tracker.md) for issue tracker commands and conventions.
- Read [docs/agents/triage-labels.md](../agents/triage-labels.md) for label mapping.
- Read [docs/agents/domain.md](../agents/domain.md) before issue or PRD work that depends on domain terminology.
